# 周计划格式规范（参考快照）

> 基于生产环境 `03-个人运营/02-计划日程/周计划-2026-第21周.md` 的格式提取。

## 优先级标记体系

| 优先级 | 标记 | 含义 | 典型场景 |
|--------|------|------|----------|
| P0 | 🔴 | 紧急修复 | 系统异常、阻断性问题、高危故障 |
| P1 | 🔴 | 核心项目推进 | 主线业务/项目任务 |
| P2 | 🟡 | 重要推进 | 日常运维、后续跟进 |
| P3 | 🟢 | 低优先级 | 备选优化、技能沉淀、版本评估 |

## 遗留问题表格格式

```
| 问题 | 严重度 | 说明 |
|------|--------|------|
| 🔴 错误码 401 | 116次 | 高频出现 |
| 🔴 备份损坏 | 系统性 | gzip 超时 |
```

## 完整文档结构（自上而下）

1. **标题行**: `# 周计划 2026 第 N 周 (MM/DD - MM/DD, 2026)`
2. **一、上周完成任务回顾**
   - ✅ 已完成任务（逐项描述，含完成时间）
   - 📊 关键成果（量化/里程碑）
   - ⚠️ 遗留问题（表格）
3. **二、本周核心目标与优先级**
   - 🔴 P0 — 紧急修复（每条含任务列表+验收标准）
   - 🔴 P1 — 核心项目推进
   - 🟡 P2 — 重要推进
   - 🟢 P3 — 低优先级
4. **三、本周待办列表**（表格）
5. **四、计划摘要**
   - 周期、本周主题、本周主要矛盾、核心里程碑、待办积压
6. **尾注**: `*自动生成 by Brain OS 主 Agent*` + 生成时间
