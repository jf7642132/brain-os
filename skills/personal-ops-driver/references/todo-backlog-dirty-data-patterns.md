## 检测方法

### 按主题相似度检测

```bash
# 提取 todo-backlog 中的问题描述，按关键词聚类
grep -E "TODO-[0-9]+-" todo-backlog.md | \
  sed 's/|.*//' | \
  sort | uniq -c | sort -rn
```

### 按来源任务检测

同一来源任务（史官）在不同日期记录同一问题：
- 检查 `来源任务` 列是否包含相同的历史文件名
- 检查 `问题描述` 是否包含相同关键词

### 按 Kanban 卡片检测

```bash
# 检查 todo 中有 Kanban 卡片 ID 但 Kanban 列表中没有的条目
# 或 Kanban 中有卡片但 todo 中没有对应条目的情况
hermes kanban list --status open
```

## 清理流程

1. **识别重复**: 按主题关键词聚类
2. **合并**: 保留最新的一条，其他标记为"已合并到 XXX"
3. **更新状态**: 已解决的问题标记 `resolved`
4. **验证**: 清理后重新统计 open/in_progress/resolved 数量

## 预防措施

1. **史官 prompt 改造**: 检查是否已存在相同主题后再记录
2. **Kanban 防重复**: 创建卡片前检查 todo 中是否已存在
3. **定期清理**: 每周运行一次脏数据检测脚本
