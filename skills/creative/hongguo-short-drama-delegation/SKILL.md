# 红果短剧分工协作改编工作流

## 工作模式
总管家（主agent）负责：
1. 统计原文字数，计算总集数（按平均150字/集，100-200字范围）
2. 按章节拆分任务，保持集数连续编号（全项目从001开始顺序编号）
3. 调度两个子agent分工协作：
   - **编剧助理**：负责改编剧本，按要求拆分分集
   - **核稿助理**：负责逐字对照原文审核，发现问题
4. 发现问题后转给编剧修正，修正后再核稿复审，通过后进入下一章
5. 用户只做最终确认，不需要子agent打扰用户

## 核心要求（红果 AI 漫剧官方规格）
- **时长**：每集约 3 分钟 AI 漫剧
- **字数**：每集严格控制在 **400-600 字汉字**（含标点，不含格式标签），根据剧情灵活调整，保留足够细节让内容不单调
- **重要提醒**：用户确认标准：签约项目每集3分钟，400-600字/集，保留更多原文细节和人物互动，项目已写到30多集，后续续写保持集数连续编号
- **格式**：`XXX 时间 内/外 地点` → 人物列表 → 画面描述 → 角色：（动作/神态）台词 → 结尾留悬念钩子 → 分隔线 `---`
- **剧情**：**严格遵循原文**，**绝对不能**添加原创剧情，不能超范围续写，必须完整保留所有核心剧情、人物互动、爽点，删除低俗过度身体描写，保持合规正能量
- **钩子**：每集结尾必须设置悬念钩子，吸引观众看下一集
- **编号**：全项目连续编号，三位数格式（001-XXX），不能重新编号，方便用户整理
- **审核**：核稿必须做到：\n  1. 逐字对照原文检查剧情完整性，无缺失无原创偏离\n  2. 检查字数是否符合 400-600字\n  3. 检查角色名字是否正确\n  4. 检查每集是否有悬念钩子\n  5. 检查剧情连贯性，跟上一集衔接是否自然\n  6. 检查合规性，无低俗内容

## 工作流步骤
```
1. 总管家：统计原文字数 → 计算该章应分集数 → 分配编号范围 → 派给编剧助理
2. 编剧助理：改编完成 → 输出所有分集文件到指定目录
3. 总管家：派给核稿助理审核 → 输出问题清单
4. 发现问题 → 总管家转给编剧助理修正 → 修正完再复审 → 直到通过
5. 全部通过 → 进入下一章
```

## 文件保存位置
**所有项目文件统一保存在wiki**：`/root/.hermes/knowledge/01-项目/02-红果短剧改编/项目名称/`

## 文件命名规则
- 原文：`chapter_XX.md` 存放在 `original/` 目录
- 剧本：`episode_XXX.md`（三位数补零）存放在 `script/` 目录
- 全项目连续编号，不按章节重新从001开始
- 当前项目《小卖部求生，我的店员全是女神》完整路径：`/root/.hermes/knowledge/01-项目/02-红果短剧改编/小卖部求生我的店员全是女神/`

## 常见问题处理
1. **全章字数超标**：重新压缩每集字数，保持总集数不变
2. **原创超范围剧情**：必须删除，重新调整分集，停在原文该章结尾处
3. **剧情缺失**：补充原文遗漏的核心信息
4. **场景错误**：统一修正回原文描述的场景

## delegate_task 调用模板
### 分派编剧任务
```json
{
  "goal": "按照红果短剧要求，将小说第N章改编成X集竖屏AI漫剧剧本，每集严格控制在400-600字（约3分钟）。",
  "context": "红果短剧官方要求：\n1. 原文路径：/path/to/original/chapter_XX.md\n2. 输出目录：/path/to/script/\n3. 集数编号：YYY-ZZZ（共X集），文件名格式：episode_XXX.md（三位数补零）\n4. **严格遵循原文内容**，不能添加原创剧情，也不能缺失核心剧情和爽点，删除低俗过度描写\n5. **每集字数严格控制在400-600字之间**（含标点），超出或不足都不合格\n6. **每集结尾必须有悬念钩子**，吸引观众看下一集\n7. 使用固定格式：XXX 时间 内/外 地点 → 人物列表 → 画面描述 → 角色：（动作/神态）台词 → 结尾悬念钩子\n8. 所有内容必须严格从原文提取，不能原创添加原文没有的剧情\n9. 集数编号必须连续正确，不能错号，保持全项目从001开始连续编号，当前已写到30多集，后续续写延续编号",
  "toolsets": ["file"]
}
```

```json
{
  "goal": "请审核《小卖部求生，我的店员全是女神》第N章改编完成的X集剧本（YYY-ZZZ）。",
  "context": "原文路径：/path/to/original/chapter_XX.md\\n剧本目录：/path/to/script/\\n审核范围：episode_YYY.md - episode_ZZZ.md（共X集）\\n\\n红果官方审核标准（必须逐条检查）：\\n1. 逐字对照原文，检查是否有剧情缺失或原创偏离（不能加原文没有的内容）\\n2. 检查角色名字是否全部正确\\n3. **检查每集字数是否严格在400-600字之间（含标点），超出或不足都不合格\\n4. **检查每集结尾是否都有悬念钩子，没有钩子不合格**\\n5. 检查集数编号是否连续正确，三位数格式是否正确，接上一集编号是否正确\\n6. 检查剧情连贯性，跟上一集结尾衔接是否自然\\n7. 检查合规性，无低俗内容\\n8. 列出所有发现的问题，必须标明集数和具体问题，没有问题也要说明\\\"全部通过\\\"\\n",
  "toolsets": ["file"]
}
```
