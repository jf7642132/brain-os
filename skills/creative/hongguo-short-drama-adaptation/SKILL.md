# 红果短剧IP改编工作流

红果短剧（字节旗下）是当前最大的短剧分账平台之一，开放给编剧入驻投稿，提供保底+分成。本技能总结网文IP改编短剧剧本的标准流程。

## 平台要求

红果短剧基本要求：
- **竖屏短剧**：每集1-3分钟，文字量100-200字/集
- **节奏要快**：第一集必须出设定、出女主、出冲突
- **每集结尾留钩子**：让观众想看下一集
- **要素齐全**：系统+求生/穿越+多女主是当前爆款公式
- **保底区间**：4万-20万 + 最高40%全生命周期分成

**用户常见偏好调整**：
- 有些用户偏好每集3分钟左右（200-300字），内容更饱满，避免过于碎片化，保证剧情连贯

## 改编原则

1. **保留核心设定，砍掉水文**：网文大段描写砍掉，只留对话、动作、冲突
2. **强化女主描写**：红果观众吃这个，每个女主出场要描写身材/气质特点
3. **冲突前置**：开篇1-3集必须把世界观+危机讲清楚
4. **每集一个小高潮/悬念**：结尾必须卡在最紧张的地方

## 开篇模板（以《小卖部求生》为例）

| 集数 | 内容要点 | 结尾钩子 |
|------|----------|----------|
| 1 | 主角穿越，规则播报 | 怪物嘶吼声响起，众人惊呆 |
| 2 | 主角观察环境，盘点物资 | 突然一声娇叱，第三个女主醒了 |
| 3 | 三个女主自我介绍完 | 门外怪物拍门，整扇门震颤 |
| 4 | 众人吓得不敢出声 | 密集拍门声像暴雨砸落，大门快撑不住 |
| 5 | 主角想起新手保护期，松口气 | 系统面板出现，提示开始绑定店主 |
| ... | ... | ... |

## 合作创业模式（常见分工）

- **甲方（入驻方）**：完成平台注册认证，挑选IP，对接平台投稿，负责沟通修改
- **乙方（改编方）**：根据IP原文改编成符合要求的短剧剧本
- **分成**：双方协商，通常甲方6-7成，乙方3-4成（乙方出内容，甲方出渠道关系）
- **长期目标**：赚到钱后升级硬件，本地部署大模型降低API成本

## 代理调度模式

使用 Hermes 子代理系统，可以创建专门的编剧助理子代理负责编写剧本，主代理只负责管理调度，不占用主代理上下文，不影响主代理同时处理其他任务：

```python
# 示例：创建项目后，分批调度子代理写作
delegate_task({
  "goal": "编写第N集剧本...",
  "context": "项目路径+用户要求...",
  "toolsets": ["terminal", "file"]
})
```

优势：
- 子代理独立上下文，专注写作不受干扰
- 可以并行多集同时创作，提高效率
- 主代理空闲处理其他任务，充分利用资源

## 注意事项

- 平台需要登录才能看IP预览，所以需要甲方复制原文内容给改编方
- 原文复制后需要保存到项目目录的 original/ 文件夹，方便随时核对角色和剧情
- 如果出现角色名字错乱，要立刻从原文核对修正，避免错误累积
- 先改编前20集给甲方看节奏，满意再继续
- 红果偏好：都市爽文、系统穿越、求生、多女主、战神、赘婿这些题材
- 用户偏好：每集3分钟左右（200-300字），内容不缺失，保证剧情连贯，不要太碎片化
