# Opportunity Solution Tree — Template

**Source**: Teresa Torres, *Continuous Discovery Habits*
**Use when**: Structuring discovery work around a specific desired outcome

## Structure

```
🎯 Desired Outcome: [single measurable outcome]
│
├── 🔍 Opportunity 1: [customer pain/need in their words]
│   ├── 💡 Solution A: [idea 1]
│   │   └── 🧪 Experiment: [hypothesis + method + metric + threshold]
│   ├── 💡 Solution B: [idea 2]
│   │   └── 🧪 Experiment: [hypothesis + method + metric + threshold]
│   └── 💡 Solution C: [idea 3]
│       └── 🧪 Experiment: [hypothesis + method + metric + threshold]
│
├── 🔍 Opportunity 2: [customer pain/need]
│   ├── 💡 Solution A:
│   │   └── 🧪 Experiment:
│   └── 💡 Solution B:
│       └── 🧪 Experiment:
│
└── 🔍 Opportunity 3: [customer pain/need]
    └── 💡 Solution A:
        └── 🧪 Experiment:
```

## Process
1. Define the desired outcome (single, measurable)
2. Map 3-7 opportunities from customer research
3. Prioritize opportunities using Opportunity Score: **Importance × (1 − Satisfaction)**
4. Generate 3+ solutions per prioritized opportunity
5. Design 1-2 experiments per promising solution
6. Include for each experiment: hypothesis, method, metric, success threshold

## Key Principles
- One outcome at a time
- Opportunities, not features ("never let customers design solutions")
- Compare and contrast (always 3+ solutions before choosing)
- Discovery is not linear — loop back if experiments fail
- Continuous, not periodic — update weekly
