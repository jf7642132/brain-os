# Product Requirements Document — Template

**Source**: phuryn/pm-skills → pm-execution → create-prd
**Use when**: Writing a product spec, documenting requirements, or reviewing an existing PRD

## 8-Section Template

### 1. Summary
(2-3 sentences: what is this about?)

### 2. Contacts
| Name | Role | Comment |
|------|------|---------|
| | | |

### 3. Background
- Context: What is this initiative about?
- Why now? Has something changed?
- Is this something that just recently became possible?

### 4. Objective
- What's the objective? Why does it matter?
- How does it align with vision and strategy?
- **Key Results** (SMART format):
  - KR1: 
  - KR2: 
  - KR3:

### 5. Market Segment(s)
- For whom are we building this?
- What constraints exist?
- (Markets defined by problems/JTBD, not demographics)

### 6. Value Proposition(s)
| Dimension | Description |
|-----------|-------------|
| **What Before** | Customer's current situation, pain |
| **How** | How product delivers solution |
| **What After** | Improved outcome/future state |
| **Alternatives** | What customers use today instead |

### 7. Solution
- UX/Prototypes (user flows)
- Key Features (detailed descriptions)
- Technology (only if relevant)
- Assumptions (what we believe but haven't proven)

### 8. Release
- What goes in v1 vs future versions?
- Avoid exact dates; use relative timeframes
- Key milestones and dependencies
